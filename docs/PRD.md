# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요

- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 흩어져 있는 개인의 다양한 링크(포트폴리오, 블로그, 유튜브, 소셜 미디어 등)를 하나의 페이지로 모아 쉽게 공유할 수 있는 서비스 제공
- **기술 및 디자인 스택 (설계 방향)**:
  - **인증**: Firebase Authentication을 이용한 구글 소셜 로그인 전용 (단일 인증)
  - **디자인 UI**: shadcn/ui 기반의 모던하고 깔끔한 디자인 시스템 적용

## 2. 핵심 기능 목록

### 2-1. 필수 기능 (MVP)
- **인증 및 회원가입**: 구글 소셜 로그인 (Firebase)
- **프로필 관리**: 닉네임 및 한 줄 소개 인라인 편집 (이미지 직접 업로드 기능 없음)
- **링크 관리**: 개별 링크 추가 및 삭제, 인라인 편집, 구글 API를 이용한 파비콘 자동 적용
- **퍼블릭 페이지 렌더링**: 닉네임 기반 URL(`/닉네임`)을 통한 공개용 링크 목록 렌더링

### 2-2. 추가 구현 기능 (Phase 2)
- **링크 클릭 통계**: 각 링크별 클릭 조회수 제공 기능

## 3. 각 기능의 상세 설명

### 3-1. 인증 및 회원가입
- **상세 내용**:
  - **오직 구글 소셜 로그인(Firebase)**만 지원하여 진입 장벽과 가입/로그인 과정을 단축시킵니다.
  - 최초 로그인 시 자동으로 회원가입 처리가 되며 기본 프로필 데이터가 생성됩니다.

### 3-2. 프로필 관리 (어드민)
- **상세 내용**:
  - **수정 방식 (인라인 편집)**: 프로필(한 줄 소개)과 닉네임 수정은 화면에서 텍스트를 바로 클릭해 수정할 수 있는 **인라인 편집(Inline Editing)** 방식을 사용합니다.
  - **URL 연동**: 닉네임은 퍼블릭 페이지 접속을 위한 URL 주소로 직접 사용됩니다. (별도의 핸들이나 `@` 기호 없이 `/닉네임` 형태 적용)
  - **이미지 업로드 없음**: 프로필 이미지 직접 업로드 기능은 제공하지 않으며, 구글 계정의 기본 프로필 이미지를 사용합니다.

### 3-3. 링크 관리 (어드민)
- **상세 내용**:
  - **링크 추가 및 구글 파비콘 API**: 목적지 URL 주소와 타이틀을 입력하여 새 링크를 생성합니다. 이때 링크 아이템의 아이콘은 **구글 파비콘 API**(`https://s2.googleusercontent.com/s2/favicons?domain={URL}`)를 호출하여 자동으로 가져와 적용합니다.
  - **인라인 편집 및 삭제**: 기존에 등록된 링크의 제목이나 URL 주소 역시 **인라인 편집**을 통해 직관적으로 바로 수정할 수 있으며, 지원하지 않는 항목은 바로 삭제할 수 있도록 제공합니다.

### 3-4. 퍼블릭 페이지
- **상세 내용**:
  - 사용자별 닉네임 기반 URL(`/닉네임`)을 통해 누구나 접속할 수 있습니다.
  - (테마 기능 없음) 별도의 테마 설정이나 컬러 커스텀 기능 없이 shadcn/ui 기본 디자인 시스템을 따르는 단일 뷰로 렌더링됩니다.
  - 모든 디바이스에 적합한 반응형 UI로 렌더링합니다.

### 3-5. 데이터베이스 모델링
- **상세 내용**:
  - 데이터 모델링 시, 사용자(User) 도큐먼트 내에 `displayName` 필드를 필수로 포함해야 합니다.
  - 각 사용자가 가지는 링크 목록 등의 데이터는 **서브 컬렉션(Sub-collection)** 방식을 활용하여 구조화합니다.

---

> **참고**: 본 PRD는 프로젝트의 구조를 잡는 기본적인 뼈대 문서입니다. 추후 작성될 **사용자 시나리오** 및 **와이어프레임** 문서를 통해 상세한 화면 흐름과 각 요소의 배치가 구체화될 것입니다.
